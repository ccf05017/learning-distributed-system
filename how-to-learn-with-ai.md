# 학습하고 동기화하기

## 배경

- AI 도구가 발달하면 어떻게 학습하는 게 좋을까?
- 학습한 결과를 어떻게 남기는 게 좋을까?
- 학습한 결과를 활용할 방법은 없을까?

## 목표

- AI 도구를 활용해서 어떻게 학습할 것인지 고민하고 실험해본다
- 학습한 결과물을 AI 도구에 어떻게 활용할지 고민하고 실험해본다

## 어떻게 학습할까?

- 기존에 문서 읽기, 책 읽기만 진행했던 내용들은 모두 다 예시 코드를 결과물로 내놓자
  - 왜 이 방법을?
    - 체감상 output 주도 학습법이 가장 효과가 좋았기 때문에 이 방법을 채택
  - 왜 굳이 AI 도구로?
    - output 주도 학습법이 효과는 좋지만, 모든 걸 코드로 결과물을 만들면 시간이 지나치게 오래 걸렸음
    - AI 도구를 활용하면 개발 속도는 큰 문제가 되지 않기 때문에 현실적인 목표수준이 됐다고 판단
  - 꼭 코드를 결과물로?
    - 인간 스스로 학습 결과물을 만들고 실행 과정을 볼 수 있는 이점이 가장 강함
    - 그 다음으로 개발자면 결과물을 기반으로 더 좋은 코드를 생산하려 할텐데, 코드 형태로 아웃풋을 만들면 AI 도구에 먹이기가 좋음
    - 학습 결과를 활용할 때 가치가 있을 거라 "추정"되어 이 방법 채택 (가설 검증 필요)
- 구체적으로 어떻게 진행할까?
  - 학습하고자 하는 내용을 학습한 뒤 어떤 범위를 어떻게 코드 결과물로 내놓을지 설계 진행
    - 이 부분은 좀 더 구체적인 방안 고민이 필요.
    - 당분간은 기존처럼 감에 의존해서 만들고 싶은 것 위주로 결과물을 생성.
  - 진행된 설계를 기반으로 AI 도구와 TDD로 페어 프로그래밍 진행 (반드시 TDD로 진행해야 함)
  - 완성된 결과물을 잘 감상(+생각)한 뒤 TDR 방식으로 구현 코드 전부 제거
  - 구현 코드를 최대한 인간의 힘으로 다시 구현
    - AI 도구를 이 과정에서 얼마나 쓰는 게 좋을지, 안 쓰는 게 좋을지 모르겠음
    - 진행하면서 효과가 어느정도인지 가설 검증 필요
  - 마무리로 AI 도구와 대화하며 학습 내용이 어떤 문제를 어떻게 풀려 했는지 핵심 요약 진행
  - 핵심 요약 내용을 바탕으로 반드시 알아야 할 유의점과 예외 상황들을 정리
  - 정리된 내용을 500자 내외의 마크다운 문서로 요약
    - 요약 내용이 이게 적정할지는 모르겠음
    - 일단 정한 글자수이기 때문에 진행하면서 유동적으로 조정 필요할 것
    - 다만, 500자 내외로 요약이 안된다면 한번에 학습하기 어려운 범위가 아니었는지 고민도 필요할 것

## 어떻게 적용할까?

- 학습한 내용을 각 AI 도구들의 특성에 맞는 맥락으로 제공하면 어떨까 추측 중
  - 강한 후보는 Claude Code의 skills 형태를 생각중
  - 다른 AI 도구들도 분명히 비슷한 무언가가 있을테니 어떻게 적용할지는 차근차근 생각해볼 것
- 제공된 내용을 바탕으로 AI 도구가 얼마나 잘 활용하는지 관찰 및 측정 필요
- 제공된 내용을 적용했을 때 얼마나 결과물이 개선되는지 관찰 및 측정 필요
- 관찰 및 측정은 어떻게?
  - 과업이 모호한 프롬프트를 주고 처리 시도 및 측정
  - 과업이 명확한 프롬프트를 주고 처리 시도 및 측정
  - 이미 구현된 코드를 기반으로 리팩토링 시도 및 측정
- 사실 이 부분은 나온 결과물 바탕으로 계속 써보면서 더 구체화해야 될 것으로 보임
- 특히 위 학습 내용이 적용이 불가능한 형태일 수 있음을 인지하고 움직일 것

## 어떻게 시작할까?

- 기존에 하려던 학습 내용들에 우선 적용해보자
  - StarRocks 공식문서 읽으면서 미니 StarRocks 만들어보기
  - 분산 시스템 설계와 구현 기법 읽으면서 패턴 구현해보기
  - GoF 디자인패턴과 함수형 프로그래밍의 교차점 탐색 및 패턴 구현해보기
- 패턴 구현해보는 부분들은 지금 생각한 방안을 쓰기가 더 용이하지 않을까?
